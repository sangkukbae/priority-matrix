# 아이젠하워 매트릭스 웹 애플리케이션 PRD (Product Requirements Document)

## 1. 개요

### 1.1 프로젝트명
**Priority Metrix (프리오리티 매트릭스)**

### 1.2 프로젝트 설명
Priority Metrix는 아이젠하워 매트릭스(Eisenhower Matrix) 기반의 웹 작업 관리 애플리케이션입니다. 사용자가 긴급성과 중요도를 기준으로 작업을 4개의 사분면으로 분류하여 우선순위를 효과적으로 관리할 수 있도록 도와줍니다.

**디자인 벤치마킹**: Trello의 깔끔한 카드 기반 UI, 부드러운 애니메이션, 직관적인 인터랙션을 참고하여 현대적이고 사용하기 쉬운 인터페이스를 구현합니다.

### 1.3 배경
미국 34대 대통령 드와이트 D. 아이젠하워의 시간 관리 철학에서 시작된 아이젠하워 매트릭스는 Stephen Covey가 "The 7 Habits of Highly Effective People"에서 대중화한 생산성 관리 도구입니다. 긴급한 업무와 중요한 업무를 명확히 구분함으로써 사용자는 가장 중요한 업무에 집중할 수 있습니다.

### 1.4 목표
- Trello 스타일의 직관적인 4분면 UI를 통한 작업 우선순위 관리
- 부드러운 드래그 앤 드롭 인터랙션
- 깔끔한 카드 디자인과 시각적 피드백
- 마감일 및 색상 코딩을 통한 시각적 우선순위 표시
- 데이터 지속화를 통한 작업 목록 영구 보존

## 2. 아이젠하워 매트릭스 개념

### 2.1 핵심 프레임워크
아이젠하워 매트릭스는 작업을 다음 두 가지 기준으로 분류합니다:
- **긴급성 (Urgent)**: 즉시 처리가 필요한 정도
- **중요성 (Important)**: 장기 목표에 미치는 영향

### 2.2 4개 사분면 구성

| | **긴급 (Urgent)** | **비긴급 (Not Urgent)** |
|---|---|---|
| **중요 (Important)** | **제1 사분면: DO (해야 할 일)**<br>즉시 처리해야 하는 작업 | **제2 사분면: PLAN (계획해야 할 일)**<br>계획적으로 처리할 작업 |
| **비중요 (Not Important)** | **제3 사분면: DELEGATE (위임할 일)**<br>다른 사람에게 위임할 작업 | **제4 사분면: DELETE (삭제할 일)**<br>제거하거나 최소화할 작업 |

### 2.3 각 사분면의 정의

#### 제1 사분면 - DO (해야 할 일)
- 긴급하면서 중요한 작업
- 마감일이 임박한 작업
- 즉각적인 주의가 필요하며 결과가 명확한 작업
- 예시: 프로젝트 마감 완료, 고객 긴급 요청 대응

#### 제2 사분면 - PLAN (계획해야 할 일)
- 중요하지만 긴급하지 않은 작업
- 장기 목표에 기여하지만 즉각적 처리가 필요 없는 작업
- 예시: 전문성 개발, 네트워킹, 개인 프로젝트 개선

#### 제3 사분면 - DELEGATE (위임할 일)
- 긴급하지만 중요하지 않은 작업
- 즉각적 처리가 필요하지만 장기 목표에 영향이 없는 작업
- 예시: 블로그 게시물 업로드, 미팅 메모 정리

#### 제4 사분면 - DELETE (삭제할 일)
- 긴급하지도 중요하지도 않은 작업
- 생산성을 저해하는 불필요한 작업
- 예시: 상태 회의, 승인 상태 공유

```css

  background-color: #FFFFFF;
  border-radius: 8px;
  box-shadow: 0 1px 0 rgba(9, 30, 66, 0.25);
  transition: all 0.2s ease;
  cursor: pointer;
}

.task-card:hover {
  background-color: #F4F5F7;
  box-shadow: 0 4px 8px rgba(9, 30, 66, 0.25);

.task-card:active {
  transform: scale(0.98);
  transition: transform 0.1s ease;
}
```

## 4. 기능 요구사항

### 4.1 핵심 기능 (Core Features)

#### 4.1.1 작업 관리 (Task Management)
| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 작업 추가 | 새 작업 생성 및 사분면 지정 | 필수 (P0) |
| 작업 편집 | 기존 작업 수정 (제목, 설명, 마감일) | 필수 (P0) |
| 작업 삭제 | 작업 제거 | 필수 (P0) |
| 작업 이동 | 드래그 앤 드롭으로 사분면 간 이동 | 필수 (P0) |
| 작업 완료 | 완료 표시 및 처리 | 필수 (P0) |

#### 4.1.2 작업 속성
| 속성 | 설명 | 필수 여부 |
|------|------|----------|
| 제목 | 작업 이름 (최소 1자) | 필수 |
| 설명 | 작업 상세 내용 | 옵션 |
| 마감일 | 완료 기한 | 옵션 |
| 중요도 플래그 | 중요 작업 표시 | 옵션 |

#### 4.1.3 사분면 관리
| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 4분면 레이아웃 | 2x2 그리드 레이아웃 | 필수 (P0) |
| 사분면 헤더 | 각 사분면의 DO/PLAN/DELEGATE/DELETE 라벨 | 필수 (P0) |
| 작업 카운트 | 각 사분면의 작업 수 배지 | 권장 (P1) |

### 4.2 UI/UX 요구사항

#### 4.2.1 Trello 스타일 카드 디자인
- 깔끔한 흰색 배경 카드
- 부드러운 모서리 (8px)
- Hover 시 미세한 그림자 증가
- 드래그 시 시각적 피드백
- 부드러운 전환 애니메이션

#### 4.2.2 헤더 디자인
- 깔끔한 미니멀리즘 디자인
- Trello Blue (#0079BF) 액션 버튼
- 다크 모드 토글

#### 4.2.3 레이아웃
- 반응형 2x2 그리드 (데스크탑)
- 1열 스택 (모바일)
- 충분한 여백과 간격
