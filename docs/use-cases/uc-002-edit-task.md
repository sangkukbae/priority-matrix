# Use Case 002: 작업 수정 (Edit Task)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **유스 케이스 ID** | UC-002 |
| **유스 케이스명** | 작업 수정 |
| **작성일** | 2026-01-10 |
| **버전** | 1.0 |
| **상태** | 초안 |

## 2. 설명

사용자가 기존 작업의 정보를 수정하는 기능입니다. 제목, 설명, 사분면, 우선순위, 마감일, 완료 상태를 수정할 수 있습니다.

## 3. 액터

| 액터 유형 | 설명 |
|-----------|------|
| **주 액터** | 일반 사용자 |
| **보조 액터** | 시스템 (로컬 스토리지) |

## 4. 사전 조건

| 조건 | 설명 |
|------|------|
| PRE-01 | 수정할 작업이 존재해야 함 |
| PRE-02 | 사용자가 수정하려는 작업을 선택해야 함 (편집 버튼 클릭) |

## 5. 사후 조건

| 조건 | 설명 |
|------|------|
| POST-01 | 작업 정보가 업데이트됨 |
| POST-02 | 작업 목록이 즉시 업데이트되어 화면에 표시됨 |
| POST-03 | updatedAt 타임스탬프가 현재 시간으로 갱신됨 |
| POST-04 | 로컬 스토리지에 변경사항이 저장됨 |

## 6. 기본 흐름

| 단계 | 액터 | 시스템 |
|------|------|--------|
| 1 | 작업 카드에서 "편집" 버튼을 클릭함 | |
| | | 2. "작업 수정" 대화상자가 열림 (기존 데이터가 미리 채워짐) |
| | | 3. 애니메이션과 함께 수정 폼이 표시됨 |
| 3 | 수정할 정보를 입력/선택함 | |
| 4 | "저장" 버튼을 클릭함 | |
| | | 5. 입력값을 검증함 |
| | | 6. 기존 Task 객체를 찾아 업데이트함 |
| | | 7. updatedAt 필드를 현재 시간으로 갱신함 |
| | | 8. Zustand 스토어를 업데이트함 |
| | | 9. 로컬 스토리지에 데이터를 저장함 |
| | | 10. 대화상자가 닫히고 화면이 업데이트됨 |
| | | 11. 수정된 작업 카드가 애니메이션과 함께 업데이트됨 |

## 7. 대체 흐름

| 대체 흐름 | 조건 | 설명 |
|-----------|------|------|
| ALT-01 | 아무 변경 없이 저장 | 변경사항이 없으면 저장 후 대화상자 닫힘 |
| ALT-02 | 제목 제거 | "작업 제목은 필수입니다" 오류 표시 |
| ALT-03 | 취소 버튼 클릭 | 수정 사항 저장 없이 대화상자 닫힘 |

## 8. 예외 상황

| 예외 상황 | 처리 방법 |
|-----------|-----------|
| EXC-01 | 작업이 삭제되어 수정 불가 | 대화상자가 닫히고 "작업이 존재하지 않습니다" 메시지 표시 |
| EXC-02 | 로컬 스토리지 오류 | 오류 로깅 후 재시도 옵션 제공 |

## 9. 트리거

| 트리거 유형 | 설명 |
|-------------|------|
| 사용자 클릭 | 작업 카드의 편집 (✏️) 버튼 클릭 |

## 10. 관련 유스 케이스

| 관련 유스 케이스 | 관계 유형 |
|-----------------|-----------|
| UC-001: 새 작업 추가 | 선행 (새 작업 후 수정 가능) |
| UC-003: 작업 삭제 | 대체 (수정 대신 삭제 가능) |
| UC-005: 작업 이동 | 대체 (사분면만 변경 시 이동 기능 사용) |
