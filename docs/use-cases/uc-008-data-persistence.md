# Use Case 008: 데이터 영속화 확인 (Verify Data Persistence)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **유스 케이스 ID** | UC-008 |
| **유스 케이스명** | 데이터 영속화 확인 |
| **작성일** | 2026-01-10 |
| **버전** | 1.0 |
| **상태** | 초안 |

## 2. 설명

사용자가 작업을 추가/수정/삭제한 후 페이지를 새로고침하거나 브라우저를 닫았다가 다시 열었을 때 데이터가 유지되는지 확인하는 기능입니다.

## 3. 액터

| 액터 유형 | 설명 |
|-----------|------|
| **주 액터** | 일반 사용자 |
| **보조 액터** | 시스템 (localStorage) |

## 4. 사전 조건

| 조건 | 설명 |
|------|------|
| PRE-01 | 하나 이상의 작업이 생성되어 있어야 함 |
| PRE-02 | 데이터가 로컬 스토리지에 저장되어 있어야 함 |

## 5. 사후 조건

| 조건 | 설명 |
|------|------|
| POST-01 | 페이지 새로고침 후에도 모든 작업 데이터가 유지됨 |
| POST-02 | 브라우저 종료 후 재실행 시에도 데이터 유지 |
| POST-03 | 로컬 스토리지에서 데이터 확인 가능 |

## 6. 기본 흐름

| 단계 | 액터 | 시스템 |
|------|------|--------|
| 1 | 작업을 추가/수정/삭제함 | |
| 2 | 페이지를 새로고침하거나 브라우저를 닫고 다시 열음 | |
| | | 3. Zustand persist 미들웨어가 로컬 스토리지에서 데이터를 로드함 |
| | | 4. 작업 목록이 이전 상태와 동일하게 복원됨 |

## 7. 예외 상황

| 예외 상황 | 처리 방법 |
|-----------|-----------|
| EXC-01 | 로컬 스토리지 데이터 손상 | 빈 상태로 시작, 콘솔에 오류 로깅 |
| EXC-02 | 스토리지 용량 초과 | 경고 메시지 표시 |

## 8. 스토리지 정보

### 로컬 스토리지 키

```
Storage Key: priority-metrix-storage
```
