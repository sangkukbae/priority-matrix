# Use Case 005: 작업 사분면 간 이동 (Move Task Between Quadrants)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **유스 케이스 ID** | UC-005 |
| **유스 케이스명** | 작업 사분면 간 이동 |
| **작성일** | 2026-01-10 |
| **버전** | 1.0 |
| **상태** | 초안 |

## 2. 설명

사용자가 드래그 앤 드롭 기능을 사용하여 작업을 한 사분면에서 다른 사분면으로 이동하는 기능입니다. 이는 아이젠하워 매트릭스의 핵심 기능으로, 작업의 긴급성과 중요성 재평가를 시각적으로 표현합니다.

## 3. 액터

| 액터 유형 | 설명 |
|-----------|------|
| **주 액터** | 일반 사용자 |
| **보조 액터** | 시스템 (@dnd-kit 라이브러리) |

## 4. 사전 조건

| 조건 | 설명 |
|------|------|
| PRE-01 | 이동할 작업이 존재해야 함 |
| PRE-02 | 드래그 앤 드롭이 지원되는 환경 (마우스/터치) |

## 5. 사후 조건

| 조건 | 설명 |
|------|------|
| POST-01 | 작업의 quadrant 필드가 새 사분면으로 변경됨 |
| POST-02 | 작업이 원본 사분면에서 제거됨 |
| POST-03 | 작업이 새 사분면에 추가됨 |
| POST-04 | 양쪽 사분면의 작업 카운트가 업데이트됨 |
| POST-05 | 로컬 스토리지에 변경사항이 저장됨 |

## 6. 기본 흐름

| 단계 | 액터 | 시스템 |
|------|------|--------|
| 1 | 작업 카드를 클릭하고 드래그 시작 | |
| | | 2. 작업 카드가 들린 형태로 표시됨 |
| 2 | 카드를 목표 사분면으로 드래그 | |
| | | 3. 목표 사분면이 하이라이트로 표시됨 |
| 3 | 카드를 목표 사분면 위에 놓음 (드롭) | |
| | | 4. 작업의 quadrant 값을 새 사분면으로 업데이트 |
| | | 5. 로컬 스토리지에 데이터 저장 |
| | | 6. 작업이 새 위치에 표시됨 |

## 7. 대체 흐름

| 대체 흐름 | 조건 | 설명 |
|-----------|------|------|
| ALT-01 | 드래그 취소 | 카드를 원래 위치에 놓으면 이동 없이 복귀 |
| ALT-02 | 잘못된 드롭 | 사분면 외부에 놓으면 원래 위치로 복귀 |

## 8. 트리거

| 트리거 유형 | 설명 |
|-------------|------|
| 마우스 드래그 | 작업 카드 클릭 후 드래그 |
| 터치 드래그 | 작업 카드 터치 후 드래그 |

## 9. 관련 유스 케이스

| 관련 유스 케이스 | 관계 유형 |
|-----------------|-----------|

| UC-004: 작업 완료 처리 | 독립 (완료된 작업도 이동 가능) |
