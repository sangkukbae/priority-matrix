# Use Case 004: 작업 완료 처리 (Complete Task)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **유스 케이스 ID** | UC-004 |
| **유스 케이스명** | 작업 완료 처리 |
| **작성일** | 2026-01-10 |
| **버전** | 1.0 |
| **상태** | 초안 |

## 2. 설명

사용자가 작업을 완료 처리하는 기능입니다. 완료된 작업은 시각적으로 구분되고, 완료 상태를 다시 토글하여 미완료로 변경할 수 있습니다.

## 3. 액터

| 액터 유형 | 설명 |
|-----------|------|
| **주 액터** | 일반 사용자 |
| **보조 액터** | 시스템 (로컬 스토리지) |

## 4. 사전 조건

| 조건 | 설명 |
|------|------|
| PRE-01 | 완료 처리할 작업이 존재해야 함 |
| PRE-02 | 작업이 아직 완료되지 않았어야 함 (미완료 상태) |

## 5. 사후 조건

| 조건 | 설명 |
|------|------|
| POST-01 | 작업의 completed 필드가 true로 변경됨 |
| POST-02 | 작업 제목에 취소선(strikethrough)이 적용됨 |
| POST-03 | 완료 체크 아이콘이 채워진 형태로 변경됨 |
| POST-04 | 로컬 스토리지에 변경사항이 저장됨 |

## 6. 기본 흐름

| 단계 | 액터 | 시스템 |
|------|------|--------|
| 1 | 작업 카드의 완료 버튼 (원형)을 클릭함 | |
| | | 2. 작업의 completed 상태를 토글함 (false → true) |
| | | 3. 작업 카드가 시각적으로 업데이트됨 |
| | | 4. 취소선이 적용되고 아이콘이 변경됨 |
| | | 5. 로컬 스토리지에 데이터를 저장함 |

## 7. 대체 흐름

| 대체 흐름 | 조건 | 설명 |
|-----------|------|------|
| ALT-01 | 이미 완료된 작업 | 완료 상태가 true → false로 변경됨 (미완료 처리) |

## 8. 예외 상황

| 예외 상황 | 처리 방법 |
|-----------|-----------|
| EXC-01 | 작업이 삭제됨 | 아무 일도 발생하지 않음 |

## 9. 트리거

| 트리거 유형 | 설명 |
|-------------|------|
| 사용자 클릭 | 작업 카드의 완료 버튼 (○ / ✓) 클릭 |

## 10. UI 요구사항

### 미완료 상태 vs 완료 상태

**미완료:**
```
┌─────────────────────────────────────┐
│ ○ │ 작업 제목                        │
└─────────────────────────────────────┘
```

**완료:**
```
┌─────────────────────────────────────┐
│ ✓ │ 작업 제목 (취소선 적용)          │
└─────────────────────────────────────┘
```
