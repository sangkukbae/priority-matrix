# Priority Metrix - Use Cases

아이젠하워 매트릭스 웹 애플리케이션의 사용자 유스 케이스 문서 모음입니다.

## 디자인 벤치마킹

**Trello Design System**을 참고하여 깔끔하고 직관적인 UI/UX를 구현합니다.

### Trello 핵심 디자인 요소

#### 색상 팔레트
| 용도 | 색상명 | HEX | RGB |
|------|--------|-----|-----|
| **주 색상** | Trello Blue | `#0079BF` | rgb(0, 121, 191) |
| **성공** | Green | `#61BD4F` | rgb(97, 189, 79) |
| **경고** | Yellow | `#F2D600` | rgb(242, 214, 0) |
| **위험** | Red | `#EB5A46` | rgb(235, 90, 70) |
| **배경** | Light Gray | `#F4F5F7` | rgb(244, 245, 247) |
| **텍스트** | Charcoal | `#172B4D` | rgb(23, 43, 77) |

#### 카드 디자인 (Trello Style)
- **배경색**: `#FFFFFF` (흰색)
- **테두리 반경**: `8px` (border-radius)
- **기본 그림자**: `0 1px 0 rgba(9, 30, 66, 0.25)`
- **Hover 그림자**: `0 4px 8px rgba(9, 30, 66, 0.25)`
- **전환 효과**: `all 0.2s ease`

#### 간격 시스템 (8px Grid)
| 간격 | 크기 | 용도 |
|------|------|------|
| xs | 4px | 아이콘 내부 여백 |
| sm | 8px | 카드 내부 패딩 |
| md | 16px | 카드 간격 |
| lg | 24px | 섹션 간격 |

## 유스 케이스 구조

각 유스 케이스 문서는 다음 구조를 따릅니다:

```
1. 개요 (유스 케이스 ID, 이름, 버전, 상태)
2. 설명 (기능에 대한 간략한 설명)
3. 액터 (주 액터, 보조 액터)
4. 사전 조건 (유스 케이스 실행 전 필요한 조건)
5. 사후 조건 (유스 케이스 실행 후 결과)
6. 기본 흐름 (주 흐름의 단계별 설명)
7. 대체 흐름 (대안 시나리오)
8. 예외 상황 (오류 처리)
9. 트리거 (유스 케이스 시작 조건)
10. UI 요구사항 (Trello 스타일 화면 레이아웃, 애니메이션 등)
11. 관련 유스 케이스 (다른 유스 케이스와의 관계)
12. 디자인 시스템 참고
```
